## v0.6 — Timed Detection Logging

This pre-release marks the next stable development milestone of the **Edge Vision Inspection System**.

v0.6 adds runtime detection logging to the smart shelf monitoring pipeline. The system now records slot-level detection results at controlled time intervals instead of logging every frame. This makes the project behave more like a real inspection/monitoring system by preserving detection evidence while avoiding excessive log generation.

### Added

* Added a CSV-based detection logger module in `src/detection_logger.py`.
* Added timed logging from the main application loop.
* Logged slot detection results at controlled intervals.
* Recorded detection data with:

  * timestamp
  * slot name
  * detection status
  * edge pixel count
  * threshold value
* Added `data/logs/` as the runtime logging location.
* Kept generated log files ignored by Git while preserving the log folder structure.
* Added v0.6 testing notes for detection logging behavior.

### Verified

* The camera/video pipeline still runs successfully.
* A1, A2, and A3 slot detection still displays correctly.
* The system still shows real-time visual overlays.
* FPS and resolution display still work.
* Detection results are written to `data/logs/detection_log.csv`.
* Logs are written periodically instead of every frame.
* Generated CSV log files are ignored by Git.
* The program exits cleanly when `q` is pressed.

### Pre-release Note

This project is still under active development before v1.0.

v0.6 improves the system by adding detection evidence logging, but the project still uses manually defined shelf regions and classical OpenCV-based image processing. It does not yet include automatic slot detection, low-stock detection, misplaced item detection, or trained AI-based object recognition.

This release should be treated as a stable development milestone, not the final completed version.
