## v0.8 — Event Evidence Capture

This pre-release marks the next stable development milestone of the **Edge Vision Inspection System**.

v0.8 adds visual evidence capture for status-change events. When a shelf slot changes state, such as `Empty → Detected` or `Detected → Empty`, the system now saves an image frame as proof of what the camera saw at the moment of the event.

This builds on the previous logging system:

* `v0.6` added timed detection logging.
* `v0.7` added status-change event logging.
* `v0.8` adds image evidence capture for those status-change events.

---

### Added

* Added event evidence capture for shelf slot status changes.
* Added `src/evidence_capture.py` to handle image saving.
* Saves processed frame images when a slot changes status.
* Stores generated evidence images under:

```text
data/evidence/events/
```

* Evidence filenames include:

  * timestamp
  * slot name
  * previous status
  * current status

Example filename:

```text
2026-06-06_14-30-10_A2_Detected_to_Empty.jpg
```

* Added `.gitkeep` support to preserve the evidence folder structure.
* Updated `.gitignore` so generated evidence images are not committed.
* Added v0.8 test notes for evidence capture validation.

### Technical Notes

v0.8 introduces a separate evidence capture module to keep the system architecture clean.

Current separation:

```text
src/shelf_config.py
= detects slot state and returns detection results

src/detection_logger.py
= writes periodic detection logs and status-change event logs

src/evidence_capture.py
= saves image evidence for status-change events

src/main.py
= coordinates camera loop, status tracking, event logging, and evidence capture
```

The evidence capture system saves the processed frame instead of the raw frame, so the saved image includes visual overlays such as slot rectangles and detection labels.

---

### Pre-release Note

This project is still under active development before `v1.0`.

v0.8 improves the system by adding visual evidence capture, but the project still uses manually defined shelf regions and classical OpenCV-based image processing. It does not yet include automatic slot detection, low-stock detection, misplaced item detection, or trained AI-based object recognition.

This release should be treated as a stable development milestone, not the final completed version.
